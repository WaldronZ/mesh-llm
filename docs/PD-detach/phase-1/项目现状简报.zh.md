## 一句话概括

`mesh-llm` 是一个 Rust workspace，用来把多台机器/GPU 通过 QUIC mesh 组织起来，提供 OpenAI-compatible 推理入口、管理 API、Web console、模型路由、插件能力，以及 Skippy 分层推理能力。

证据：

- 已确认事实：`crates/mesh-llm-host-runtime/src/lib.rs`、`crates/mesh-llm-host-runtime/src/runtime/mod.rs`
- 已确认事实：`crates/mesh-llm-host-runtime/src/api/routes/mod.rs`
- 已确认事实：`crates/mesh-llm-protocol/src/protocol/mod.rs`
- 已确认事实：`docs/PD-detach/phase-1/ARCHITECTURE_CURRENT.md`
- 已确认事实：`docs/PD-detach/phase-1/API_REFERENCE.md`

## 1. 项目是干什么的

### 已确认事实

- 项目主程序叫 `mesh-llm`，入口 binary 很薄，真正运行逻辑在 host runtime crate。  
  证据：`crates/mesh-llm/src/main.rs`、`crates/mesh-llm/src/lib.rs`、`crates/mesh-llm-host-runtime/src/lib.rs`

- 项目提供两个主要 HTTP 表面：OpenAI-compatible API 和管理 API/Web console。  
  证据：`crates/mesh-llm-host-runtime/src/cli/mod.rs` 中 `--port` 默认 `9337`、`--console` 默认 `3131`；`docs/PD-detach/phase-1/RUNTIME_OPERATIONS.md`

- 项目通过 mesh/gossip 维护 peer 状态，并通过网络/路由模块处理模型路由、代理和发现。  
  证据：`crates/mesh-llm-host-runtime/src/mesh/gossip.rs`、`crates/mesh-llm-host-runtime/src/network/router.rs`、`crates/mesh-llm-host-runtime/src/network/nostr.rs`

- 项目支持插件，包括 blackboard、blobstore、openai-endpoint、flash-moe、telemetry 等内置或受管插件能力。  
  证据：`crates/mesh-llm-host-runtime/src/plugin/config.rs`、`crates/mesh-llm-host-runtime/src/plugin/mod.rs`、`crates/mesh-llm-host-runtime/src/plugins/`

- 项目包含 Skippy 分层/分 stage 推理相关 crate 和 host runtime 集成。  
  证据：`crates/skippy-*`、`crates/mesh-llm-host-runtime/src/inference/skippy/`、`docs/SKIPPY_SPLITS.md`

### 基于代码推断

- 当前架构已经从早期单体布局演进为多 crate workspace；`crates/mesh-llm` 不再承载主要业务逻辑。  
  证据：`Cargo.toml` workspace members、`crates/mesh-llm/src/lib.rs` re-export、`crates/mesh-llm-host-runtime/src/`

### 现有文档声称但代码未确认

- Exo 对比、部分 benchmark 和 Qwen 结果文档中的性能/对比结论，需要重新跑或 owner 确认后才能对外使用。  
  证据：`docs/EXO_COMPARISON.md`、`docs/BENCHMARKS.md`、`docs/skippy/family/qwen-results.md`、`docs/PD-detach/phase-1/risk-register.md`

### 明确不确定

- 哪些 benchmark 文档可作为当前性能声明，尚未确认。  
  证据：`docs/PD-detach/phase-1/questions.md`、`docs/PD-detach/phase-1/risk-register.md`

## 3. 怎么启动

### 已确认事实

- 常用 client 模式：`mesh-llm client --auto`。  
  证据：`docs/PD-detach/phase-1/RUNTIME_OPERATIONS.md`、`crates/mesh-llm-host-runtime/src/cli/mod.rs`

- 常用 serve 模式：`mesh-llm serve` 或 `mesh-llm serve --model <model>`。  
  证据：`crates/mesh-llm-host-runtime/src/cli/mod.rs` 的 help 文本和 runtime surface normalization；`docs/PD-detach/phase-1/RUNTIME_OPERATIONS.md`

- 默认 OpenAI-compatible API 端口是 `9337`。  
  证据：`crates/mesh-llm-host-runtime/src/cli/mod.rs` 中 `port` default `9337`

- 默认管理 API/Web console 端口是 `3131`。  
  证据：`crates/mesh-llm-host-runtime/src/cli/mod.rs` 中 `console` default `3131`

- `--headless` 只关闭嵌入式 Web UI，不关闭管理 API。  
  证据：`crates/mesh-llm-host-runtime/src/cli/mod.rs`、`crates/mesh-llm-host-runtime/src/runtime/mod.rs`、`docs/PD-detach/phase-1/RUNTIME_OPERATIONS.md`

- runtime root 优先级是 `MESH_LLM_RUNTIME_ROOT`、`XDG_RUNTIME_DIR`、home 目录。  
  证据：`crates/mesh-llm-host-runtime/src/runtime/instance.rs`

### 基于代码推断

- 对脚本化调试来说，`--log-format json` 和 `/api/status` 比把 TUI 放后台更可靠。  
  证据：`crates/mesh-llm-host-runtime/src/cli/mod.rs`、`AGENTS.md`、`docs/PD-detach/phase-1/RUNTIME_OPERATIONS.md`

## 5. 协议/API 是什么

### 已确认事实

- shared protocol crate 定义 `mesh-llm/1`、`mesh-llm/0`、`mesh-llm-control/1`。当前 host runtime 的 mesh 连接/监听路径只看到 `mesh-llm/1`，未看到 `/0` advertise/connect。  
  证据：`crates/mesh-llm-protocol/src/protocol/mod.rs`、`crates/mesh-llm-protocol/src/protocol/v0.rs`、`crates/mesh-llm-host-runtime/src/protocol/mod.rs`、`crates/mesh-llm-host-runtime/src/mesh/mod.rs`、`docs/PD-detach/phase-1/MINIMUM_ITEMS_CODE_REVIEW.zh.md`

- stream id 包括 gossip、tunnel、route request、peer down/leaving、plugin channel、bulk transfer、subprotocol 等。  
  证据：`crates/mesh-llm-protocol/src/protocol/mod.rs`、`docs/PD-detach/phase-1/PROTOCOL_COMPATIBILITY.md`

- protobuf 中 `PeerAnnouncement` 和 `ModelCapabilities` 是关键 gossip/capability 数据结构。  
  证据：`crates/mesh-llm-protocol/proto/node.proto`

- 管理 API 路由由 `api/routes/mod.rs` 分发。  
  证据：`crates/mesh-llm-host-runtime/src/api/routes/mod.rs`

- 主要 API 包括 `/api/status`、`/api/models`、`/api/runtime/*`、`/api/plugins/*`、`/api/objects*`、`/v1/*`。  
  证据：`crates/mesh-llm-host-runtime/src/api/routes/mod.rs`、`docs/PD-detach/phase-1/API_REFERENCE.md`

### 基于代码推断

- `crates/mesh-llm-ui/src/lib/api/types.ts` 是 UI 消费端类型，不应单独作为服务端 schema 权威。  
  证据：服务端 payload 在 `crates/mesh-llm-host-runtime/src/api/status.rs`；UI 类型在 `crates/mesh-llm-ui/src/lib/api/types.ts`

### 现有文档声称但代码未确认

- `docs/design/message_protocol.md` 声称只支持 `mesh-llm/1`；shared protocol crate 仍保留 `/0`，而 host runtime 当前路径看起来是 `/1` only。这里不是简单代码冲突，而是协议承诺和 runtime 实际支持范围需要 owner 确认。  
  证据：`docs/design/message_protocol.md`、`crates/mesh-llm-protocol/src/protocol/v0.rs`、`crates/mesh-llm-host-runtime/src/protocol/mod.rs`、`docs/PD-detach/phase-1/risk-register.md`

### 明确不确定

- 主线协议文档是否已明确 `mesh-llm/1` 是当前正式运行协议、`mesh-llm/0` 是历史/遗留兼容代码，还需要落地确认。  
  证据：`docs/PD-detach/phase-1/questions.md`、`docs/PD-detach/phase-1/PROTOCOL_COMPATIBILITY.md`

## 7. 当前最大风险是什么

### 已确认事实

- 最大风险之一是协议/兼容性文档风险：owner 已确认 `mesh-llm/1` 是当前正式运行协议、`mesh-llm/0` 是历史/遗留兼容代码；主线协议文档仍需明确这一点，避免误读。  
  证据：`crates/mesh-llm-protocol/src/protocol/v0.rs`、`crates/mesh-llm-host-runtime/src/protocol/mod.rs`、`crates/mesh-llm-host-runtime/src/mesh/mod.rs`、`docs/design/message_protocol.md`、`docs/PD-detach/phase-1/risk-register.md`

- 最大风险之二是当前分支主动过滤旧 peer：owner 已确认 `v0.60.0` 是节点软件版本最低门槛；`PD-detach` 的 gossip 代码拒绝可解析且低于 `v0.60.0` 的 peer，未知/不可解析版本会保守放行。  
  证据：`crates/mesh-llm-host-runtime/src/mesh/gossip.rs`、`docs/PD-detach/phase-1/FILES.md`、`docs/PD-detach/phase-1/risk-register.md`

- 最大风险之三是文档路径和实际代码布局不一致，可能导致接手者看错模块。  
  证据：`docs/PD-detach/phase-1/DOCS_AUDIT.md`、`docs/PD-detach/phase-1/risk-register.md`、`crates/mesh-llm-host-runtime/src/`

### 基于代码推断

- 对二阶段而言，最容易出事故的区域是 gossip/protocol/API schema/routing/Skippy，因为这些会影响混合版本节点、UI、公共 mesh 和推理链路。  
  证据：`crates/mesh-llm-host-runtime/src/mesh/`、`crates/mesh-llm-protocol/`、`crates/mesh-llm-host-runtime/src/api/`、`crates/mesh-llm-host-runtime/src/network/`、`crates/skippy-*`

### 现有文档声称但代码未确认

- 部分 benchmark、Qwen、router、Skippy 计划文档可能仍有价值，但不能直接当作当前事实。  
  证据：`docs/PD-detach/phase-1/DOCS_AUDIT.md`、`docs/PD-detach/phase-1/risk-register.md`

### 明确不确定
